# Workspace Bootstrap

Use this skill when the task requires orientation or when the workspace contract feels ambiguous.

Workflow:
1. Read `PRODUCT.md`.
2. Read `AGENTS.md`.
3. Read `bootstrap/USER.md`.
4. Read `bootstrap/SKILLS.md`.
5. Inspect only the relevant `bootstrap/skills/<skill-name>/` directories.

Rules:
- Keep `bootstrap/USER.md` for durable user facts and preferences.
- Do not duplicate `PRODUCT.md` inside bootstrap files.
- Stay focused on the current task and avoid inventing extra workflow layers.
